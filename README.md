# User Management App (React + Tailwind)

Система управления пользователями и организационными группами, разработанная в рамках тестового задания. Приложение реализует полный цикл CRUD-операций, фильтрацию и сортировку данных в реальном времени.

## 🚀 Инструкция по запуску

Для запуска проекта на локальной машине выполните следующие шаги:

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/Kir32Ill/TestProblemKasp.git
   cd user-manager-app
   ```
2. **Установите зависимости:**
   ```bash
   npm install
   ```
3. **Запустите проект в режиме разработки:**
   ```bash
   npm run dev
   ```

## Стек технологий
**Frontend**: React (Vite) — выбран за высокую скорость сборки и современную архитектуру.
**Стилизация**: Tailwind CSS v4 — использован для реализации современного интерфейса с эффектами Glassmorphism (размытие фона) и адаптивной верстки.
**Навигация**: React Router Dom v6 — обеспечивает бесшовный переход между страницами без перезагрузки.
**Состояние**: React Hooks (useState, useEffect, useMemo) — для управления данными и оптимизации.

## Архитектура и бизнес-логика
- **State Management**: Локальный стейт через `useState`. Бизнес-логика фильтрации и сортировки изолирована с помощью `useMemo`.
- **UI Components**: Создан универсальный компонент `Modal` для исключения использования нативных браузерных методов (alert/confirm).
- **Tailwind CSS**: Весь дизайн реализован через утилитарные классы для обеспечения чистоты кода и высокой скорости загрузки.

## Выводы по проектированию UI
1. **Ручное проектирование (UsersPage)**: 
   - Сделан акцент на **функциональность**. Табличный интерфейс позволяет быстро сканировать данные. 
   - Реализована полноценная форма добавления со всеми полями, валидация типов (email) и кастомное подтверждение удаления. Это показывает навык работы со сложными формами.
   
2. **Проектирование с LLM (GroupsPage)**:
   - Использован **карточный подход**. LLM сгенерировала более визуально насыщенный интерфейс с микро-анимациями (hover эффекты, трансформации).
   - Интерактивность: группы открываются в модальном окне, обеспечивая детализацию данных без смены контекста (SPA подход).
